ECHOFRAMES — Development Roadmap
Phase 0 — Product Definition (1–3 days)

Before writing more code, finalize the core product constraints.

Define the rules of the platform

Decide:

• Maximum photo distance detection (ex: 30m–50m radius)
• Maximum echoes per user per day
• Photo storage limit per echo
• Comment limits / moderation rules
• Whether echoes can expire

Define discovery model

Echo discovery may be:

Option A — Automatic

App checks location in background

Push notification when near echo

Option B — Map Based

User opens map and sees nearby echoes

Recommendation for MVP

Start with Map-based discovery only.
Background GPS + push notifications is expensive and complex early.

2. PostGIS Setup

Enable extension:

CREATE EXTENSION postgis;

Echo location column:

location GEOGRAPHY(POINT, 4326)

Nearby query example:

SELECT *
FROM echoes
WHERE ST_DWithin(
    location,
    ST_MakePoint(user_lon, user_lat)::geography,
    50
);

This returns echoes within 50 meters.

Phase 3 — Core Logic

Estimated time: 3–5 days

Echo Ranking Algorithm

Avoid old echoes dominating.

Basic ranking formula:

score =
(upvotes - downvotes)
+ recency_boost
+ randomness

Example:

score =
rating_score
+ (1 / age_in_days)
+ random(0,0.3)

Calculate inside:

Supabase Edge Function

Echo Hotspots

Biggest Technical Risks
1 — Location accuracy

GPS error can be 5–20m.

You must allow radius detection not exact point.

2 — Storage costs

Photos can become expensive.

Solutions:

compress images
limit resolution
limit uploads
3 — Notification spam

If poorly designed users get:

50 notifications walking in city

You need:

cooldowns
MVP Definition (Very Important)

The real MVP should only include:

• login
• map
• take photo
• upload echo
• view echo
• comments

No messaging, no push notifications yet.

That alone is enough to test the concept.x
